Nova هو تطبيق Next.js قياسي. يمكنك نشره في أي مكان يعمل فيه Next.js.

## Vercel (نقرة واحدة)

هذا هو المسار الأسرع. اضغط زر deploy في [README](https://github.com/badhope/nova)، اعمل fork للمستودع، واملأ مفتاح LLM API واحدا على الأقل أثناء الإعداد.

يبني Vercel التطبيق مع كل commit ويقدّم static assets تلقائيا من edge network.

## Docker

يتضمن المستودع `Dockerfile` جاهزا للإنتاج. ابنِ وشغّل:

```bash
docker build -t nova .

docker run -p 3000:3000 \
  -e OPENAI_API_KEY=sk-... \
  nova
```

للإعدادات متعددة الحاويات (مع reverse proxy أو خدمات أخرى بجانبه)، استخدم `docker-compose`:

```yaml title="docker-compose.yml"
services:
  nova:
    image: nova:latest
    ports:
      - '3000:3000'
    env_file:
      - .env.local
    restart: unless-stopped
```

## الاستضافة الذاتية على VM

ابنِ وابدأ باستخدام `pnpm`:

```bash
pnpm install
pnpm build
pnpm start      # يستمع على port 3000 افتراضيا
```

ضع nginx أو Caddy أمامه لإنهاء TLS. للصفوف ذات الزيارات العالية، فكّر في تشغيل عدة app instances خلف load balancer — Nova عديم الحالة افتراضيا (الحالة محفوظة في IndexedDB داخل المتصفح).
